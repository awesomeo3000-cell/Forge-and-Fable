## Behavior corrected

1. **Fixed HP never asks to roll** — LevelUpModal seeds hpGained from fixedHpGain(), marks step complete immediately, shows "+X HP (fixed)" label, no roll button.
2. **Fixed creation fills hpRolls** — buildStartingHp() populates hpGains array with identical fixed gains per extra level; characterPayload() always persists them.
3. **Level-down uses shared reversion** — handleLevelDown() calls revertHpLevel(): recorded gains removed deterministically; fixed-legacy chars without history use the fixed-gain fallback; rolled/manual legacy chars without history are refused with a user-visible reason.
4. **Current HP preserved on level-down** — min(oldCurrentHp, newMaxHp) preserves damage; 0 HP stays 0 HP (no accidental resurrection).
5. **Manual input max clamped** — Math.max(1, hitDie + conMod) ensures the input never has max < min when CON is negative.
6. **Empty rolled input stays incomplete** — buildStartingHp() returns first-level HP only when rolled rolls are empty/undefined; does not fabricate fixed gains.
