## Batch 1: Quick Wins & Bug Fixes
**Scope:** Data cleanup + small independent bug fixes. Low risk, no type changes.

### #10 Remove dead `dnd-master-data.json`
- Delete `src/data/dnd-master-data.json` (198 KB, never imported)

### #12 Fix double ROLLBACK in campaign store
- **File:** `src/lib/campaignStore.ts` ~line 437-454
- The inner try/catch does `ROLLBACK` then re-throws, and the outer catch does `ROLLBACK` again on the already-rolled-back transaction. Fix: remove the inner `ROLLBACK` + re-throw, let the outer catch handle it. Or restructure to a single try/catch with `ROLLBACK` in one place.

### #13 Campaign sync error feedback
- **File:** `src/components/ForgeAndFableApp.tsx` ~line 482
- Replace the empty `catch {}` with a counter for consecutive failures. After 3+ consecutive failures, show a status message like "Campaign sync interrupted — check your connection." Reset the counter on any successful sync. This avoids spamming on transient blips but alerts the user on sustained outages.

### #9 Fix spells missing `classes` field
- **File:** `src/data/spells.json`
- ~19 spells (Tasha's summons, psionic talents) lack a `classes` array. Add the correct class lists based on spell school and source. E.g., `summon-lesser-demons` → ["Sorcerer", "Warlock", "Wizard"], `mind-sliver` → ["Sorcerer", "Warlock", "Wizard"], etc.

### #15 Use class IDs for spell→class matching
- **File:** `src/data/spells.json` + `src/lib/spells.ts` + `src/lib/spellsClient.ts`
- Currently `spellsForClass()` matches by display name (e.g., `"Bard"`). Add a `classIds` array to spell data (or a mapping) and filter by `classId` instead of `name`. This survives any future name changes. The `classes` display-name array is kept for backward compat; a new `classIds: string[]` field is the source of truth.

## Batch 4: Dice & Vision
**Scope:** `RollDrawer.tsx`, `HeroSheet.tsx` senses section, `utils.ts`.

### #24 Custom dice formula parser
- **utils.ts:** Add `parseDiceFormula(formula: string)` function that handles:
  - Basic: `2d6`, `1d8+3`, `4d6kh3` (keep highest 3)
  - Multiple groups: `2d6+1d4+3`
  - Modifiers: `+N` or `-N`
  - Returns `{ groups: { sides: number, count: number, keepHighest?: number }[], modifier: number, error?: string }`
- **utils.ts:** Add `rollFormula(parsed)` that rolls each group, applies keep-highest, sums, adds modifier.
- **RollDrawer.tsx:** Add a text input field for custom formula. Parse on Enter or button click. Show error for invalid formulas. Roll with the dice overlay animation.

### #27 Vision/sense overlap handling
- **HeroSheet.tsx senses section:**
  - Collect all sense entries from active effects that match patterns like `/darkvision (\d+)/i`, `/truesight (\d+)/i`, `/blindsight (\d+)/i`
  - When multiple of the same sense type exist (e.g., Darkvision 60 ft from race + Darkvision 120 ft from effect), display only the highest value
  - Parse `race.speed`-style range numbers and compare, keeping the max per sense type
