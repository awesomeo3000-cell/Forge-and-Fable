## Part 1: Readability — darker text, higher contrast

### 1a. Fix the missing base text color on `.cs-sheet` (globals.css ~line 3304)
- **Problem:** `.cs-sheet` never sets `color`, so inherited text from `<body>` is light `#ece0c8` on a light `#ece1c9` paper background. Elements only work because each sets its own `color` explicitly.
- **Fix:** Add `color: var(--ink);` to the `.cs-sheet` rule block. This matches what `.paper-surface` and `.campaign-panel` already do.

### 1b. Darken `--ink-3` for secondary text (globals.css ~line 64)
- **Problem:** `--ink-3: #8a7c60` gives ~3:1 contrast on paper — below WCAG AA. Used for spell metadata (`.cs-spell-card span`), ability labels, muted notes, and sense rows.
- **Fix:** Darken to `#5e503a` (~5:1 contrast). Inside `.cs-sheet` this is the effective value; the `:root` value at line 18 stays `#16120c` (already set from prior work).

### 1c. Increase base font size floor (globals.css ~line 5606)
- **Problem:** The `clamp(16px, 0.45vw + 12px, 20px)` at line 3328 is overridden by `calc(1em * var(--sheet-scale, 1))` at line 5606. With `--sheet-scale: 1` and body at 17px, the effective base is 17px — readable but the `em`-multiplied children get small fast (0.68em = 11.6px).
- **Fix:** Add `min-font-size` protection on key small text elements. Specifically: bump `.cs-spell-card span`, `.cs-feature-card p`, `.cs-save-name`, `.cs-vital-hp-max` from `0.68–0.88em` to `0.8–0.92em` minimum. Add `font-size: 0.82em` floor to `.cs-muted`.

## Part 2: Wizard spellbook as a 5th reference tab

### 2a. Add "spellbook" to RefTab type (HeroSheet.tsx ~line 68)
```ts
type RefTab = "features" | "traits" | "spells" | "spellbook" | "inventory";
```
Add to `REF_TABS` array:
```ts
{ id: "spellbook", label: "Spellbook" },
```

### 2b. Conditional visibility — wizard only (HeroSheet.tsx ~line 420–424)
The Spellbook tab shows only when `canManageSpellbook` is true (wizard with spell list). Add it to `visibleTabs` filtering:
```ts
const visibleTabs = REF_TABS.filter((t) => {
  if (t.id === "spells") return hasSpellTab;
  if (t.id === "spellbook") return canManageSpellbook;
  return true;
});
```
When a wizard has both Spells and Spellbook tabs: **Spells** tab shows prepared/castable spells with slot pips; **Spellbook** tab shows the two-panel library + prepared manager.

### 2c. Two-panel Spellbook tab layout (HeroSheet.tsx, new JSX after the spells panel ~line 1447)
Add a new `<div>` for `refTab === "spellbook"`:

**Left panel — Library (all known spells, searchable + alphabetized):**
- Search input (`cs-spell-search`)
- Learn manager: `<select>` dropdown + "Add to Spellbook" button (moved from current spells panel)
- Alphabetized spell list grouped by level, each spell with a "Forget" button
- Spells sorted alphabetically within each level

**Right panel — Prepared list + slot tracker:**
- Prepared count: "Prepared X / Y"
- Slot pips per level (reused from current spells panel)
- List of prepared spells with unprepare toggle

Layout via CSS grid: `grid-template-columns: 1fr 1fr` on desktop, single column on mobile.

### 2d. CSS for the two-panel layout (globals.css, new rules)
```css
.cs-spellbook-panel {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: 12px;
}
@media (max-width: 760px) {
  .cs-spellbook-panel { grid-template-columns: 1fr; }
}
```

### 2e. Remove wizard manager from Spells tab
When the wizard has a dedicated Spellbook tab, the Spells tab no longer shows the Learn/Forget manager (it stays on non-wizard Spells tabs). The Spells tab for wizards shows only castable spells with slot pips and prepared markers.

## Part 3: Searchable, alphabetized spell list (both tabs)

### 3a. Add search state (HeroSheet.tsx, near line 545)
```ts
const [spellSearch, setSpellSearch] = useState("");
```

### 3b. Add search input to both Spells and Spellbook panels


### 3d. CSS for search input (globals.css)
Style `.cs-spell-search` consistently with other sheet inputs (`.cs-glass-btn`-like border, `var(--rule)` border, paper background).

## Part 4: Custom item form — match effect form styling

### 4a. Add missing CSS definitions (globals.css, near the effect form rules ~line 6801)
The item form classes `.cs-inv-form` and `.cs-inv-form-actions` have **zero CSS**. Add them mirroring the effect form pattern:

```css
.cs-inv-form {
  margin-top: 8px;
  display: flex;
  flex-direction: column;
  gap: 7px;
}
.cs-inv-form input,
.cs-inv-form select {
  font-family: var(--font-body);
  font-size: 0.85em;
  padding: 6px 10px;
  border: 1px solid var(--rule-soft);
  border-radius: var(--r-sm);
  background: var(--ground-3);
  color: var(--parchment);
}
.cs-inv-form input:focus,
.cs-inv-form select:focus {
  outline: 2px solid var(--select);
  border-color: var(--select);
}
.cs-inv-form-actions {
  display: flex;
  gap: 6px;
}
```

This gives the item form the same gap spacing, input borders, focus rings, and font sizing as the effect form — making them visually consistent.
