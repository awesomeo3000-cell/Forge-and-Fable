## Campaign Roll Reliability & UI Repair — Implementation Plan

### Phase A: Enrolled character + immediate merge (critical functional fixes)

**A1. Fix enrolled character resolution (§7):**
- In `handleCampaignRollRequest`, look up the enrolled character from `campaignSync.members` (matching `user.id`) instead of using `selected`.
- If the enrolled character is found in the vault (`characters` array), use it for modifiers. If not found, show "Campaign character unavailable" and disable the Roll button.
- Display `Rolling as <characterName>` in the notification card.
- Apply the same fix to `applyCampaignRest`.

**A2. Immediate roll merge (§11):**
- Change `postCampaignRollApi` from `void` to return the created `CampaignRollRow`.
- In `recordHistory`, `await` the API call and merge the returned roll into `campaignSync.rolls` immediately (deduped by id on next sync).
- On failure, show "Roll completed locally but could not be shared" status.

### Phase B: Rich notification card + UI fixes

**B1. Structured notification card (§6, §15):**
- Create `describeRollRequest()` in `rollRequest.ts` returning `{ title, prompt, meta, buttonLabel }`.
- Rewrite `campaign-event-card` rendering in `CampaignPanel.tsx` to show:
  1. Title (e.g., "PERCEPTION CHECK" in `--font-label` caps)
  2. Prompt (DM's free-text, if any)
  3. Meta line: "DC 15 · Advantage" (or "Normal roll")
  4. "Rolling as <name>"
  5. Timestamp
  6. Action buttons
- Fix the `strong` color inheritance: use `var(--ink)` instead of `var(--parchment)`.
- Add `.campaign-notification-title`, `.campaign-notification-prompt`, `.campaign-notification-meta` CSS classes with explicit ink colors.

**B2. Normalize Roll/Dismiss button sizing (§16):**
- Add `min-height: 36px` to `.campaign-event-actions button`.
- Make Roll button label descriptive: "Roll" / "Roll with advantage" / "Roll with disadvantage" based on `describeRollRequest().buttonLabel`.
- Add `disabled` state during roll execution to prevent double-submission.
- Style Dismiss as secondary (transparent bg, visible border, same height).

**B3. Campaign action buttons (§13-14):**
- The list-view "New Campaign" and "Join" buttons already use `dj-btn dj-btn-primary` which has `background: var(--ink); color: var(--paper); border: 1px solid var(--ink)`. This is a dark-on-light parchment control already — not a black rectangle.
- The Join form's Cancel (`glass-button`, 46px min-height) vs Join (`dj-btn-primary`, no min-height) disparity: add `min-height: 38px` to `.campaign-form-actions button` for consistent sizing.
- Verify both buttons in the running browser before claiming fixed.
