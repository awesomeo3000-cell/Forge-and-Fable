## Priority 1: Expand the Encounter Combatant Model

### Overview
Replace the minimal `InitiativeCombatant` with a richer `CampaignCombatant` type. Player combatants store references (not a duplicate sheet); NPC/enemy combatants carry their full state in the encounter.

### 1. Type definitions (`src/types/campaign.ts`)
Add `CampaignCombatant` and `CampaignCombatantCondition` types. Update `InitiativeState.combatants` to `CampaignCombatant[]`. Keep `InitiativeCombatant` as a deprecated alias.

### 2. Server-side validation (`src/lib/campaignStore.ts`)
Extend `clampInitiativeState` with all new fields (kind, memberUserId, characterId, currentHp, maxHp, tempHp, ac, hidden, defeated, concentratingOn, conditions[], privateNote, statBlock). Backward-compat: normalize old `isPlayer`→`kind:"player"`, `hp:{current,max}`→flat fields.

Update `rollCampaignInitiative` to set `kind:"player"`, `memberUserId`, `characterId`. Strip DM-private data (`privateNote`, NPC `conditions`) from player sync payload in `visibleInitiative`.

### 3. API routes
Minimal changes: `roll` route passes `characterId` through to the store.

### 4. Client app + API
Pass `characterId` when submitting initiative rolls. Type alignment throughout.

### 5. DM Table Panel (`DMTablePanel.tsx`)
Add `kind` selector to add-combatant form. Update HP editing to flat `currentHp`. Player combatant rows: read-only HP/AC from member summary. NPC rows: editable. Show `defeated`, `conditions` chips, `concentratingOn`.

### 6. Tests
Update campaign v2 tests to new shape. Add test for backward compat normalization. Add test that privateNote/conditions are stripped from player sync.

### Key decisions
- **Single source of truth**: no separate encounter array — combatants live in `InitiativeState.combatants`
- **Player HP/AC derived**: from member summary at sync time, not stored in encounter (stores `memberUserId`/`characterId` references only)
- **NPC state persists**: DM-editable HP/AC/conditions in the encounter JSON, survives refresh
- **Lightweight conditions**: `{ id, label, advantageMode?, stack? }` — display/tracking aid, not full effects engine
- **Backward compat**: old-format combatants normalized on read via `clampInitiativeState`
