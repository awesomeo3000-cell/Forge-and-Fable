## Script 1: `build_chimaerus_log.py` (Easiest — existing mythic template)

**Source:** `WoWCombatLog-070826_170232.txt` lines 20388–227209 (mythic kill, 313.5s)

1. Extract fight segment + header (COMBAT_LOG_VERSION, zone/map changes)
2. Remove Murraydeek's UNIT_DIED event (died at 2m24s)
3. Continue Murraydeek's damage rotation through the full 313.5s by duplicating his damage pattern from his alive period, scaled to fill remaining duration
4. Scale Murraydeek's total damage to ~58M (185k DPS target)
5. Reduce other 18 players' damage proportionally to keep total raid damage close to original
6. Shift all timestamps from 7/8/2026 → 7/13/2026
7. Update COMBATANT_INFO for Murraydeek to reflect new stats if needed
8. Output: `WoWCombatLog-071326_XXXXXXXX.txt`

## Key Technical Details

### Damage Scaling Formula
```
scale_factor = target_dps / current_dps
Murraydeek_new_damage = Murraydeek_old_damage × scale_factor
Others_new_damage = Others_old_damage × (1 - (Murraydeek_delta / total_raid_damage))
```
This preserves total raid damage while hitting Murraydeek's target.

### Player GUID Replacement
Each heroic player gets mapped to exactly one mythic player. Five heroic players get dropped. Their damage events get redistributed to similar-role mythic players.

### Timestamp Shifting
All timestamps shifted: `7/8/2026 17:02:32.623` → `7/13/2026 21:00:00.000` (base offset)
Fight-specific timestamps preserve relative timing within the encounter.

### Ability Breakdown Validation
After processing, verify damage composition matches WCL 95th percentile parses:
- LV: Necrotic Coil (13.7%), Rider's Champion (11.1%), Army of the Damned (11.0%), Lesser Ghoul (10.8%), etc.
- CC: Dread Plague (13.9%), Lesser Ghoul (11.6%), Rider's Champion (11.2%), etc.
- Chimaerus: Dread Plague (14.6%), Lesser Ghoul (12.3%), Rider's Champion (11.9%), etc.

### File Output
All three logs output to `E:\World of Warcraft 11\_retail_\Logs\` with randomized filename suffixes (e.g., `WoWCombatLog-071326_HHMMSS.txt`).

## What This Plan Does NOT Do

- **Does NOT create mythic-specific boss abilities** — LV and CC use heroic boss patterns labeled as mythic. This is a known limitation but produces valid parseable logs.
- **Does NOT pull data from WCL events API for reconstruction** — Using existing log templates is more reliable for format correctness.
- **Does NOT modify healing data** — Healing events are preserved as-is unless redistribution is needed for raid balance.
