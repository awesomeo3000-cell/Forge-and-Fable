## Portrait Selector Modal & Party Integration — Implementation Plan

### Current State vs Target

| Feature | Current | Target (Plan) |
|---|---|---|
| Portrait selector | Inline gallery in AppearancePanel only | Modal dialog accessible from creator + sheet |
| Labels on portraits | Ancestry + presentation shown | No visible labels — image-only tiles |
| Gender/presentation filter | "Masculine" / "Feminine" buttons | Removed entirely |
| Ancestry filter | Dropdown with ancestry names | "Suggested" / "All Portraits" tabs (no ancestry labels visible) |
| Creator integration | No portrait field | PortraitField in Step 0 (Setup) of CreatorPanel |
| Selection state | Direct onChange | Temporary pending state with Save/Cancel |
| Component architecture | Monolithic PortraitPicker + embedded CharacterPortrait | Structured: CharacterPortrait, PortraitField, PortraitSelectorModal |
| Portrait catalog IDs | `tiefling-female`, `human-male` | Opaque IDs: `portrait-tiefling-01`, `portrait-human-02` |
| DM Party portraits | Working (uses CharacterPortrait in CharacterStateVisuals) | Keep working, extract to shared component |

---

### Implementation Phases

#### Phase 1: Clean up portrait data layer (`src/data/portraits.ts`)

1. **Rename type** — Remove `label` and `presentation` fields from `PortraitOption`. Replace `ancestry: string` with `suggestedAncestries: string[]` (plan spec). 
2. **Use opaque IDs** — Rename IDs: `tiefling-female` → `portrait-tiefling-02`, `human-male` → `portrait-human-01`, etc. Map to existing filenames.
3. **Update helpers** — `PORTRAIT_BY_ID`, `suggestPortraitAncestry`, `ANCESTRY_LIST` all remain internal only.
4. **Update tests** — `tests/portraitCatalog.test.ts` to match new shape.

#### Phase 2: Extract shared `CharacterPortrait` component

1. **Create** `src/components/portraits/CharacterPortrait.tsx` — extract from `CharacterStateVisuals.tsx`:
   - Props: `portraitId?`, `characterName`, `size?`, `shape?`, `decorative?`, `className?`
   - Resolves portrait via catalog lookup by ID
   - Falls back to class art, then initials
   - Handles broken images
   - No internal metadata exposed
2. **Update** `CharacterStateVisuals.tsx` — re-export or re-use the shared component
3. **Update** `PartyRail.tsx` and `CharacterInspector.tsx` — update imports

#### Phase 3: Build `PortraitSelectorModal` 

1. **Create** `src/components/portraits/PortraitSelectorModal.tsx`:
   - Uses `useFocusTrap` hook (existing pattern) + `role="dialog"` + `aria-modal="true"`
   - Desktop: split layout — scrollable grid (left), large preview (right)
   - Mobile: stacked layout — preview on top, grid below
   - Two tabs: "Suggested" (filtered by `suggestPortraitAncestry(draft.raceId)`) and "All Portraits"
   - No ancestry labels, no presentation/gender labels, no portrait names visible
   - Tiles are image-only circular thumbnails (80-96px desktop, 72-84px mobile)
   - Selected state: gold ring + checkmark badge + `aria-pressed="true"`
   - Neutral accessible labels: "Portrait option 1", "Portrait option 2", etc.
   - Temporary `pendingPortraitId` state separate from saved value
   - Cancel restores previous; Save commits pending
   - Escape key = Cancel, close button = Cancel, outside click disabled
   - Return focus to trigger on close
2. **Props**: `open`, `onClose`, `onSave`, `value` (current portraitId), `suggestedAncestry?`

#### Phase 4: Build `PortraitField` 

1. **Create** `src/components/PortraitField.tsx`:
   - Compact row: circular portrait preview (or initials) + "Choose Portrait" / "Change Portrait" button
   - Opens `PortraitSelectorModal` on click
   - Used in CreatorPanel Step 0 and AppearancePanel

#### Phase 5: Integrate into CreatorPanel 

1. **Update** `CreatorPanel.tsx` Step 0 (Setup):
   - Add `PortraitField` below the name input
   - Add `portraitUrl` to the draft via `onDraftChange`
2. **Update** `QuickbuilderPanel.tsx`:
   - Add `PortraitField` in the final name entry step
   - Include `portraitUrl` in the draft passed to `onComplete`
3. **Update** `createInitialDraft()` in `utils.ts` — ensure `portraitUrl: ""` persists (already present)

#### Phase 6: Update AppearancePanel 

1. Replace inline `PortraitPicker` gallery with new `PortraitField` + modal
2. Remove the "Choose from gallery" toggle button
3. Keep the URL text input for external image URLs (supports custom uploads)
4. Keep the portrait preview span using the shared `CharacterPortrait` component

#### Phase 7: Remove old PortraitPicker (or repurpose)

1. Either delete `src/components/PortraitPicker.tsx` or keep as internal-only debug tool
2. Remove associated CSS in `globals.css` for `.portrait-picker-*` classes, replace with modal styles

#### Phase 8: Add modal CSS 

1. Add styles in `globals.css` for:
   - `.portrait-modal` — overlay and dialog container
   - `.portrait-modal-grid` — scrollable grid
   - `.portrait-modal-tile` — image-only circular tiles
   - `.portrait-modal-preview` — large preview panel
   - `.portrait-modal-tabs` — "Suggested" / "All Portraits" tabs
   - Responsive breakpoints for mobile stacking
   - Match Forge & Fable's existing dark/parchment aesthetic

#### Phase 9: Validation & edge cases 

1. Update `validateCharacter.ts` — already validates `portraitUrl` format; ensure catalog IDs are validated (not arbitrary paths)
2. Handle: no portrait, invalid ID, broken image, ancestry change, imported characters

#### Phase 10: Verify & test 

1. `npm run lint` 
2. `npm run build`
3. `npm test` (update `portraitCatalog.test.ts` and `dmTableParty.test.ts`)
4. Manual verification: new character, existing character, save, cancel, escape, keyboard nav, mobile
